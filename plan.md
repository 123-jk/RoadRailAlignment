# JOSM 道路/铁路平面线形插件开发计划

## 当前实现状态

已完成可试用 V1：

- Maven 构建
- JOSM 插件入口、工具按钮和右侧参数面板
- 可浮动独立参数窗口，进入地图模式时自动打开
- 道路 / 铁路 Way 输出
- 两点直线
- 三点 PI 圆曲线，可选两端缓和曲线
- 大角度圆曲线/回环，可画超过 180 度的弯道并可附加整圈
- 可按当前 R 推荐缓和曲线长度
- 三点圆曲线可跨模式复用上一段直线作为入切线
- 保留三点缓和曲线近似采样代码，但主 UI 隐藏该模式
- 实时预览
- 中文界面
- 既有 Way 接入点吸附
- 从空状态绘制直线/三点圆曲线时，可吸附既有源线条并复用其直线方向
- 两节点既有线参与源线方向吸附时按直线处理，不再触发局部曲率估算崩溃
- 匝道模式可在插件模式内点击识别并选中既有 Way
- 接入点切线、近似里程、点击偏移、局部半径和曲率估算
- 单端切线接出匝道
- 两条选中 Way 的双端 G1 连接
- 双端连接自动 R/缓和优化，尽量取最大可用 R
- 双端连接支持圆弧-圆弧-圆弧候选，并可按附加整圈数强制加入回环
- 双端连接支持按所在线方向/反向所在线判断接入方向，用于灯泡线/回头线
- 匝道按既有线曲率加入缓和过渡，支持多段曲线和直线混合
- 单端既有线接出匝道生成后保留末端切线，支持继续作业
- 单端匝道缓和段按目标点左/右侧决定切出方向，避免曲线段切出时回咬原线
- 已有节点识别与吸附，生成时复用被吸附节点
- 生成 Way 时端点/控制锚点按容差复用已有节点，中间采样点只做极小误差复用
- 最小半径检查
- 三点圆曲线按实际转角计算
- 连续作业，生成后可保留终点作为下一条线起点
- 连续绘制直线和三点圆曲线时支持上一段末端切线延伸吸附
- 连续模式下三点圆曲线只生成到曲线终点，保留出切线方向给下一段直线
- Esc 清空当前控制点、预览和连续状态
- 回撤一步，撤回最后一个控制点并同步恢复匝道/连续状态
- 主下拉框隐藏单独的缓和曲线近似模式，优先通过三点圆曲线/匝道启用缓和曲线
- 几何烟雾测试脚本

V1 明确不做完整 CAD 优化求解、纵断面、横断面、超高和施工图。

## 目标

开发一个 JOSM 插件，用于辅助绘制道路、铁路及互通匝道的平面线形。插件只处理平面线形，不处理纵断面、横断面、超高、结构层和施工图。

最终输出应为 JOSM 原生对象：

- `Node`
- `Way`
- 常规 OSM 标签

## 插件定位

插件名称暂定：

```text
RoadRailAlignment
```

核心能力：

- 基于控制点生成道路/铁路中心线
- 支持直线、圆曲线和缓和曲线近似
- 从既有 JOSM Way 提取接入点的切线方向和估算曲率
- 生成与既有道路/铁路切线连续的匝道线形
- 预览线形并生成普通 OSM Way

第一版优先保证可用，不追求完整工程设计软件能力。

## 非目标

暂不做：

- 纵断面设计
- 横断面设计
- 超高计算
- 铁路轨道结构设计
- 桥梁、隧道、涵洞设计
- 施工图出图
- 完整 CAD 替代功能

## 开发阶段

### 阶段 1：插件骨架与基础绘线

目标：让插件能在 JOSM 中加载，并能生成最简单的道路/铁路 Way。

任务：

- 建立 Maven 插件项目
- 创建插件入口类 `RoadRailPlugin`
- 创建自定义绘图模式 `AlignmentMapMode`
- 在工具栏添加插件按钮
- 支持地图点击获取控制点
- 支持两点生成直线 Way
- 支持选择道路或铁路类型
- 自动添加基础标签

道路默认标签：

```text
highway=road
source=road_rail_alignment_plugin
```

铁路默认标签：

```text
railway=rail
source=road_rail_alignment_plugin
```

验收：

- 插件能被 JOSM 加载
- 点击两点能生成一条 Way
- 生成结果可被 JOSM 撤销/重做

### 阶段 2：普通平面线形

目标：支持 PI 法生成直线 + 圆曲线。

任务：

- 建立内部线形模型
- 支持多个 PI 控制点
- 支持为交点设置圆曲线半径
- 计算切点、圆心、圆曲线方向和曲线长度
- 按采样间距生成节点
- 支持预览线形
- 支持撤销上一控制点

核心模型：

```text
Alignment
AlignmentElement
StraightElement
CircularArcElement
AlignmentPoint
```

验收：

- 三个及以上控制点可以生成带圆曲线的 Way
- 半径不足或几何不成立时给出提示
- 生成节点密度可控

### 阶段 3：既有道路/铁路接入点识别

目标：能够从既有 JOSM Way 上提取接入状态。

接入点状态包含：

```text
P: 坐标
T: 切线方向
K: 曲率估算值
Side: 左侧/右侧接入
SourceWay: 来源 Way
Station: 沿 Way 的近似里程
```

任务：

- 支持选择既有 Way
- 支持在 Way 附近点击选择接入点
- 将点击位置投影到 Way 上
- 取接入点前后一定距离的局部点列
- 估算切线方向
- 用局部点拟合圆曲线，估算半径和曲率
- 显示识别置信度

第一版识别策略：

- 局部方向变化很小：认为是直线，曲率为 0
- 局部点可拟合稳定圆：认为是圆曲线
- 其他情况：只使用切线方向，曲率标记为不可靠

验收：

- 点击既有 Way 能得到接入点坐标和切线方向
- 曲线段能给出大致半径
- 识别结果显示在插件面板中

### 阶段 4：互通匝道 G1 衔接

目标：生成与既有道路切线连续的匝道。

先支持：

- 出口匝道
- 入口匝道
- 两条既有道路之间的连接匝道

线形组合优先支持：

```text
直线 + 圆曲线
圆曲线 + 直线
圆曲线 + 直线 + 圆曲线
```

约束：

- 起点位置连续
- 起点切线方向连续
- 终点位置连续，可选终点切线方向连续
- 半径不小于用户设置的最小半径
- 采样间距受控

新增模型：

```text
TieInPoint
RampAlignment
RampConstraints
RampSolver
```

验收：

- 可以从一条既有 Way 切线接出一条匝道
- 可以连接两条既有 Way，并尽量满足两端切线方向
- 不可解时说明原因，而不是生成明显错误线形

### 阶段 5：缓和曲线与 G2 衔接

目标：支持更适合道路/铁路工程线形的曲率连续过渡。

任务：

- 添加 `SpiralElement`
- 实现 clothoid 或近似缓和曲线采样
- 支持从既有曲率 `K0` 过渡到目标曲率 `K1`
- 支持缓和曲线 + 圆曲线 + 缓和曲线
- 匝道接主线圆曲线时尽量保持曲率连续

验收：

- 直线接圆曲线可通过缓和曲线过渡
- 主线圆曲线接匝道时可指定是否强制 G2 连续
- 生成线形的方向和曲率变化平滑

### 阶段 6：互通匝道候选方案

目标：面向互通设计场景给出多个候选线形。

支持类型：

- 直接式匝道
- 半定向匝道
- 环形匝道
- S 形匝道

任务：

- 根据起终接入点、侧向和半径约束生成多个候选方案
- 对候选方案进行评分
- 评分指标包括长度、最小半径、反向曲线数量、与既有道路冲突
- 支持用户切换预览候选方案

验收：

- 至少能为常见两路相交场景生成两个以上候选
- 用户能选择其中一个方案生成 Way

## 几何原则

- 所有距离、角度和曲率计算都在 JOSM 当前投影坐标系下完成
- 不直接用经纬度计算工程距离
- 内部先生成平面坐标点，再转换为 JOSM `LatLon`
- 曲线离散应同时考虑最大采样间距和最大弦高误差
- 生成节点数量应有上限保护

## 测试计划

优先做纯几何单元测试，减少对 JOSM UI 的依赖。

测试内容：

- 两点直线采样
- PI 法圆曲线计算
- 左转/右转圆曲线
- 半径过小错误
- 弦高误差控制
- 局部直线切线估算
- 局部圆曲线半径拟合
- 匝道起点切线连续
- 两端切线连续求解失败时的错误提示

## 风险

- JOSM API 版本变化可能影响插件加载
- 既有 OSM Way 往往是折线，不一定能可靠反推出真实工程线形
- 匝道自动求解存在不可解场景，必须明确提示
- 曲率连续 G2 比切线连续 G1 复杂，不能放进第一版 MVP
- 节点过密可能影响 JOSM 编辑性能

## 当前最小可执行目标

先完成以下闭环：

1. 插件能加载
2. 有一个绘图按钮
3. 点击两点生成直线道路/铁路 Way
4. 点击三个点生成一段圆曲线连接线形
5. 可以从既有 Way 上选一个接入点并获得切线方向
6. 可以从该接入点切线接出一条简单匝道

完成这个闭环后，再加入缓和曲线和更复杂的互通候选方案。
