English version: [README.en.md](README.en.md)

RoadRailAlignment 在 JOSM 中按控制点生成道路、铁路、匝道和连接线的平面线形。插件输出 JOSM 原生 `Node` / `Way`，只处理平面线形，不处理纵断面、横断面、超高或施工图。

## 主要功能

- 地图绘制模式和独立参数窗口
- 默认 `智能基础线形` 模式：普通画线、既有线接出匝道、连接两条既有线都在同一个模式里完成
- 直线、PI 圆曲线、带缓和曲线的圆曲线、大转角圆曲线和回环线
- 单端匝道、两端既有线连接、两端连接自动优化半径和缓和段长度
- 连续作业：生成一段后保留终点、切线和必要的匝道曲率，继续画下一段
- 既有节点吸附和端点复用
- 道路、匝道、干线、铁路和高铁常用标签预设
- 中文、英文、法文、德文、西文界面资源

## 基础模式怎么画

默认绘制模式是 `智能基础线形`。这个模式会根据点击位置自动判断要画什么：

1. 第一点击在空白处：创建新起点。因为没有既有线切线，第一段只能画直线。
2. 第一点击靠近既有 `Way`：吸附为匝道接入点，第二点击空白处会生成单端匝道。
3. 第一点击靠近既有 `Way`，第二点击靠近另一条既有 `Way`：生成两端切线连续的连接线。
4. 开启 `连续作业` 后，上一段终点会作为下一段起点继续使用。

旧的专用模式仍然保留：

- `两点直线`
- `三点圆曲线（含缓和曲线）`
- `大转角圆曲线 / 回环`
- `从既有线接出匝道`
- `连接两条既有线`

## 匝道连续曲率

开启 `连续作业` 和 `保留匝道曲率给下一段` 时，单端匝道默认按下面的形态生成：

```text
0 曲率 / 上一段曲率 -> 入口缓和 -> 曲线 -> 出口缓和 -> 0 曲率
```

如果下一段匝道与上一段往相同方向弯曲，插件会在下一段生成成功后回改上一段：

```text
上一段：0 曲率 / 上一段曲率 -> 入口缓和 -> 曲线
下一段：上一段末端曲率 -> 入口缓和 -> 曲线 -> 出口缓和 -> 0 曲率
```

这样连续同向匝道不会在段间硬缓和回 0，也能画出回转匝道。若下一段反向弯曲或不是连续匝道，则上一段仍保留默认的出口缓和到 0。

## 参数说明

- `采样间距`：生成节点的间距，数值越小节点越密。
- `曲线/最小半径`：曲线和匝道使用的半径约束。
- `缓和曲线长度`：启用缓和曲线时的过渡段长度。
- `使用缓和曲线`：为曲线和匝道加入缓和段。
- `保留匝道曲率给下一段`：连续同向匝道复用上一段末端带符号曲率。
- `连续作业`：生成后保留终点，方便继续画下一段。
- `吸附已有节点` / `节点吸附容差`：生成时优先复用既有节点。
- `两端连接自动优化半径/缓和段`：自动估算两端连接可用的半径和缓和段。
- `生成后回填优化参数`：把自动优化结果写回参数面板。
- `附加整圈数`：给大转角曲线或两端连接增加完整回环。
- `接入方向`：控制两端既有线连接的切线方向，可选自动、沿源线、反向源线。

## 快捷键

- `Esc`：清空当前控制点和预览。
- `Backspace`：撤销上一个控制点。
- `Ctrl+Z`：在插件模式中优先撤销上一个控制点。

## 构建

要求：

- JDK 11+
- Maven
- `lib/josm-tested.jar`

下载 JOSM tested jar：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\tools\download-josm.ps1
```

编译并打包插件：

```powershell
mvn package
```

输出文件：

```text
target\RoadRailAlignment.jar
```

备用构建脚本：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\build.ps1
```

## 测试

运行几何烟雾测试：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\tools\run-geometry-smoke.ps1
```

也可以运行 Maven 测试流程：

```powershell
mvn test
```

## 安装

把 `target\RoadRailAlignment.jar` 放到 JOSM 插件目录，然后重启 JOSM：

```text
%APPDATA%\JOSM\plugins
```

## 注意事项

- 生成的 `Way` 只写对象类型对应的 OSM 标签，不写插件标识 tag。
- 既有线接入依赖当前可编辑数据层。
- `按 R 推荐` 的缓和曲线长度只是面向制图的经验值，来自按半径比例估算并做上下限限制与取整的简化规则。它用于让 JOSM 中生成的线形更平顺、外观更接近真实，不替代工程设计；真实缓和曲线需要结合设计速度、横向加速度变化率、超高及其渐变、规范限值等更多参数计算。
- 较小的 `采样间距` 会提高曲线近似精度，但直线段也可能生成较多不必要节点。实际绘图中建议结合 JOSM 的简化路径功能（`Shift+Y`）一起使用，在保留曲线精度的同时减少直线段节点数量。
- 自动优化是绘图辅助，不是完整工程 CAD 求解器。
